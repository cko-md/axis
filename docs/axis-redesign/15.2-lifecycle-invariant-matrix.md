# Wave 15.2 Convergence — Lifecycle Invariant Matrix (analysis only)

**Scope:** reconcile the lifecycle authority of `codex/vector-arcade-envoys` (PR #236, head `a72f3da7`)
against `codex/phase9-resume-executor` (PR #195, head `84cc5ca6`). Read-only; no writes to shared branches.

**Central finding:** the two branches implement the same lifecycle *concepts* with **disjoint function
name sets** (0 overlap). Names not colliding is *worse* than colliding: both migrations could be applied
without a SQL error, leaving **two contradictory authorities live at once**. phase9 is a conceptual
superset (adds WebAuthn/passkey authority, explicit idempotent creation, scope-completeness, stale-approval
expiry, and the browser-DML lockdown guards). **Canonical = phase9. Drop VECTOR `202607161000_lifecycle_claims`.
Rewire VECTOR call-sites.**

Legend for **Decision**: `CANONICAL=phase9` (keep phase9 RPC, rewire VECTOR) · `PORT` (rewire VECTOR call-site) ·
`GAP` (VECTOR lacks it — phase9 fills it) · `AUDIT` (confirm no VECTOR-unique behavior lost before dropping).

## Reconciliation rules applied
1. **Retain the strongest tested behavior** → phase9 wins every row where it is a strict superset (5,6,7,8,14) and every
   row where it splits an atomic concern into an explicitly named CAS/idempotent RPC (1–4, 9–13).
2. **Retain matching regression tests** → keep phase9's lifecycle SQL contract + concurrent-CAS suites as canonical; VECTOR's
   arcade CAS suites are retained for the *arcade* tables only (see migration reconciliation), not for lifecycle.
3. **Record the canonical migration/RPC** → phase9 `1300/1302/1400/1401`.
4. **Reject duplicate/contradictory authority** → **drop VECTOR `202607161000_lifecycle_claims.sql`.** Do not apply both.

## Mandatory pre-drop audits (do during the port slice, not now)
- **Row 12** `recover_stale_routine_approval_resume`: byte-compare recovery/quarantine semantics against phase9's
  `expire_stale_approvals` + fencing-token reclaim in `claim_routine_resume`. If VECTOR encodes a recovery path phase9
  lacks, port that behavior into a phase9-named function *before* dropping.
- **Row 2 / Row 10**: confirm phase9's `create_*_with_activity` and `start_claimed_routine_step` provide the atomicity
  and idempotency VECTOR relied on.
- **Row 14**: VECTOR's `guard_routine_resume_claim` trigger is a claim guard, not the browser-write lockdown. The lockdown
  is phase9's contract migration `202607161401` and stays BLOCKED until the compatible production SHA is live.
